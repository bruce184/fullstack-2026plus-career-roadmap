# Job Application Strategy

## Before Applying

- GitHub profile should clearly communicate the target role.
- Pinned repositories should have strong READMEs.
- CV should be one page, English, ATS-friendly, and backed by evidence.
- Interview notes should cover frontend, backend/API, database, testing, security, DevOps, mobile, and system design.

## Application Tracking

Use `career/job-tracker/` to track companies, roles, dates, status, links, and follow-up notes.

## CV Rule

Only claim skills and projects that have evidence in repositories, logs, screenshots, demos, tests, or documentation.
