# Portfolio Plan

## Positioning

Software Engineering student building toward full-stack development with web, mobile, backend, database, testing, and AI-integrated project experience.

## GitHub Profile Checklist

- Clear headline and target role.
- Short technical focus summary.
- Links to the parent roadmap and selected child repositories.
- No excessive decoration.
- Pinned repositories should represent real evidence.
- Profile README draft: `docs/github-profile-readme-draft.md`.

## Evidence Types

- Source code
- README files
- Screenshots or demos
- Weekly logs
- Interview notes
- Architecture diagrams
- Tests or CI runs
- CV bullet backlog

## Pinned Repo Direction

Pin only repositories that show useful evidence. Unpin forks or old coursework unless they clearly support the full-stack story.
