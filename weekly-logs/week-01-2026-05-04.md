## Request-flow Diagram Draft

```mermaid
sequenceDiagram
    participant User
    participant Client as Web/Mobile Client
    participant API as Backend API
    participant Service as Business Logic
    participant DB as Database

    User->>Client: Clicks, submits form, or opens screen
    Client->>API: HTTP request with method, route, headers, body
    API->>API: Validate input and auth
    API->>Service: Run use case
    Service->>DB: Read or write data
    DB-->>Service: Query result
    Service-->>API: Domain result
    API-->>Client: JSON response with status code
    Client-->>User: Render loading, success, error, or updated UI
```
